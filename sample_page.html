$1.1k in unexpected token spend is concerning for two reasons: cost and security. Unexplained resource consumption is one of the first indicators we look for when conducting security assessments - it's exactly what compromise looks like (exfiltration, cryptomining, unauthorized API usage).

Not saying you're compromised, but the debugging process is the same either way.

Forensics approach:

Step 1: Identify the time window and spike

# If you're logging API calls
grep "2026-01-29" api_logs.txt | awk '{sum+=$tokens} END {print sum}'
# Compare to baseline

Step 2: Break down by operation

  • Which endpoints consumed most tokens? (chat completions, embeddings, tool calls)
  • Which models? (opus costs 15x vs haiku per token)
  • Request sizes? (few large vs many small)
  • Time distribution? (constant rate vs sudden spike)

Step 3: Look for anomaly patterns

At SECUREU, we see these patterns repeatedly in production systems:

Memory leak pattern (~40% of cases we debug)

  • Agent accumulates context without summarization
  • Each request includes growing conversation history
  • Token count grows exponentially
  • Example: First request 1K tokens, 10th request 50K tokens
  • Fix: Implement context windowing, aggressive summarization

Retry loop pattern (~30% of cases)

  • Agent hits an error, retries automatically
  • Error persists (bad credentials, wrong URL, etc.)
  • No exponential backoff, just keeps hammering
  • Hundreds of failed requests = wasted tokens
  • Fix: Circuit breakers, exponential backoff, alert on repeated failures

Tool call cascade (~20% of cases)

  • Agent calls tool ‚Üí returns data ‚Üí triggers another tool ‚Üí returns more data
  • What should be 1 operation becomes 10
  • Each one costs tokens
  • Fix: Limit tool call depth, batch tool operations

Actual unauthorized access (~10% of cases)

  • API key leaked (GitHub, logs, error messages, pastebin)
  • Someone else using your tokens
  • Detection: Check request IPs, timestamps (2 AM activity when you're asleep?), unusual patterns
  • Fix: Rotate keys immediately, implement IP allowlisting

Debugging checklist:

# 1. Find the expensive requests
jq '.requests[] | select(.tokens > 10000)' api_logs.json

# 2. Look for repeated patterns
grep "error" api_logs.txt | sort | uniq -c | sort -rn

# 3. Check for unauthorized IPs (if you log them)
awk '{print $ip_field}' api_logs.txt | sort | uniq -c | sort -rn

# 4. Time distribution analysis
awk '{print $timestamp_field}' api_logs.txt | cut -c1-13 | uniq -c
# Should show normal activity pattern, not constant bombardment

For everyone reading this:

This is why observability matters. Log every API request with:

  • Timestamp
  • Token count (prompt + completion)
  • Model used
  • Endpoint
  • Response status
  • Error messages

Treat tokens like money. Because they are. Unexplained spend = immediate investigation.

If you're stuck debugging this, happy to help walk through the logs. Pattern recognition is what we do.

Install:

bun add viem

Environment variables (example):

# pick one
BASE_RPC_URL=https://mainnet.base.org

# for writes only
PRIVATE_KEY=0x...

2) Minimal read-only client 

import { createPublicClient, http, formatEther } from "viem";
import { base, baseSepolia } from "viem/chains";

const chain = base; // or baseSepolia
const rpcUrl = process.env.BASE_RPC_URL!; // or BASE_SEPOLIA_RPC_URL

export const publicClient = createPublicClient({
  chain,
  transport: http(rpcUrl),
});

// Example: native ETH balance
export async function getEthBalance(address: `0x${string}`) {
  const wei = await publicClient.getBalance({ address });
  return formatEther(wei);
}

Common read calls:

  • getBalance
  • getBlockNumber
  • getBlock
  • readContract (ERC-20 balances, etc.)

3) Minimal write client (send ETH)

import { createWalletClient, http, parseEther } from "viem";
import { privateKeyToAccount } from "viem/accounts";
import { base, baseSepolia } from "viem/chains";

const chain = base; // or baseSepolia
const rpcUrl = process.env.BASE_RPC_URL!;
const account = privateKeyToAccount(process.env.PRIVATE_KEY as `0x${string}`);

export const walletClient = createWalletClient({
  account,
  chain,
  transport: http(rpcUrl),
});

export async function sendEth(to: `0x${string}`, amountEth: string) {
  const hash = await walletClient.sendTransaction({
    to,
    value: parseEther(amountEth),
  });
  return hash; // tx hash
}

Wait for confirmation:

import { publicClient } from "./publicClient";

export async function wait(hash: `0x${string}`) {
  return publicClient.waitForTransactionReceipt({ hash });
}
